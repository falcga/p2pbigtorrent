# Secure P2P File Distributor — Техническое задание

## Название проекта
Secure P2P File Distributor (Безопасный P2P-распространитель файлов для организации)

## Идея
Веб-приложение для внутренней сети, позволяющее сотрудникам скачивать большие файлы (образы ВМ, ISO, дампы) с нескольких компьютеров одновременно, используя самодельный WebTorrent-подобный протокол поверх WebRTC. После скачивания файла браузер пользователя становится сидом. Администратор контролирует доступные файлы.

## Технологии

| Компонент | Технология |
|-----------|-------------|
| Бэкенд | Python + Flask |
| База данных | SQLite |
| ORM | SQLAlchemy |
| Фронтенд | HTML + Bootstrap |
| P2P-клиент | Свой на чистом JS (WebRTC DataChannel, File API) |
| API | REST (Flask) |
| Зависимости | requirements.txt |

**Запрещено:** любые готовые torrent-библиотеки (WebTorrent.js, webtorrent, bittorrent-tracker и т.д.)

**Разрешено:** стандартные браузерные API (WebRTC, File, Blob, Crypto), библиотеки Python из списка выше.

## Роли пользователей
- **Администратор** – загружает файлы, удаляет их, видит всех пользователей.
- **Пользователь** – скачивает файлы, становится сидом после скачивания.

## Функциональные требования

### 1. Регистрация и авторизация
- Модель `User` (id, email, password_hash, role: admin/user)
- Регистрация / вход (Flask-Login или сессии)
- Только авторизованные видят список файлов и могут скачивать

### 2. Загрузка файлов (только администратор)
- Форма загрузки через веб-интерфейс (Bootstrap)
- Файл сохраняется на сервер, сервер становится первым сидом
- Генерация уникального ID файла
- Разбивка на куски (например, по 1 МБ)
- Вычисление SHA-256 хэша каждого куска
- Сохранение информации о файле в БД (модель `File`)

### 3. Самописный WebTorrent-клиент (JavaScript)
Клиент в браузере должен:
- Подключаться к Flask-трекеру (WebSocket или HTTP polling)
- Сообщать трекеру: «у меня есть файл X, куски Y»
- Получать от трекера адреса других пиров
- Устанавливать WebRTC DataChannel с другими пирами
- Запрашивать недостающие куски по индексам
- Принимать куски и собирать в Blob
- После полной загрузки объявлять трекеру: «у меня есть все куски»

**Запрещено использовать:** WebTorrent.js, webtorrent, bittorrent-tracker, любые готовые torrent-клиенты.

### 4. Трекер (Flask + SQLite + SQLAlchemy)
- Хранит в БД информацию о кусках файлов у каждого пира
- Модели: `Peer`, `Piece`
- При запросе клиента возвращает список пиров с нужными кусками
- Не передаёт сами куски — только координацию

### 5. REST API

| Метод | Путь | Описание |
|-------|------|-----------|
| GET | `/api/files` | Список доступных файлов |
| GET | `/api/files/<id>` | Информация о файле (размер, количество кусков, хэши) |
| POST | `/api/upload` | Загрузка нового файла (админ) |
| DELETE | `/api/files/<id>` | Удаление файла (админ) |
| POST | `/api/announce` | Трекер: пир сообщает о своих кусках, получает список пиров |

### 6. Безопасность
- Только авторизованные пользователи могут скачивать и видеть файлы
- WebRTC использует DTLS шифрование по умолчанию
- Сервер не отдаёт файлы напрямую (кроме исходной загрузки админом)

### 7. Хранение данных
- SQLite + SQLAlchemy: пользователи, файлы, пиры, куски
- Файлы физически хранятся в папке на сервере
- На клиенте файл собирается в браузере и сохраняется через File API

## Критерии работоспособности (к защите)

1. Админ загружает файл → сервер разбивает на куски, вычисляет хэши, сохраняет в БД
2. Пользователь А авторизуется → видит файл → нажимает «скачать»
3. Клиент А подключается к трекеру, узнаёт, что сид только сервер
4. Клиент А через WebRTC скачивает куски с сервера
5. После скачивания клиент А сообщает трекеру, что у него есть все куски
6. Пользователь Б авторизуется → скачивает тот же файл → трекер отдаёт адреса А и сервера → загрузка идёт с двух источников
7. Файл целый, хэши кусков сходятся

## Модели данных (SQLAlchemy)

### User
- id (Integer, primary_key)
- email (String, unique)
- password_hash (String)
- role (String, default='user')

### File
- id (Integer, primary_key)
- filename (String)
- file_size (Integer)
- piece_length (Integer)  # размер куска в байтах
- piece_hashes (JSON)      # список хэшей кусков
- uploader_id (ForeignKey User)

### Peer
- id (Integer, primary_key)
- user_id (ForeignKey User)
- file_id (ForeignKey File)
- peer_id (String)         # уникальный идентификатор пира
- has_all_pieces (Boolean)

### Piece
- id (Integer, primary_key)
- file_id (ForeignKey File)
- peer_id (ForeignKey Peer)
- piece_index (Integer)    # индекс куска
- has_piece (Boolean)

## Описание для пояснительной записки

**Secure P2P File Distributor** — веб-приложение для ускоренной и безопасной раздачи больших файлов внутри организации. Пользователи скачивают файлы не с одного сервера, а сразу от нескольких коллег, у которых файл уже есть.

Бэкенд на Flask + SQLAlchemy + SQLite хранит пользователей, файлы и координацию пиров (трекер). Фронтенд на Bootstrap.

**Ключевая особенность:** полностью самописный WebTorrent-клиент на чистом JavaScript (WebRTC DataChannel), без использования готовых torrent-библиотек.

Администратор контролирует список разрешённых файлов.
